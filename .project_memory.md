# Project Memory: Stock_Price_Calculator

> Global memory: before starting or continuing this project, read `/Users/hannah/Documents/Codex/GLOBAL_MEMORY.md` first.


## Project Overview
- **Name**: Stock_Price_Calculator
- **Purpose**: A desktop application (Electron + HTML/CSS/JS) to calculate stock target prices based on intrinsic values.
- **Tech Stack**: Electron, Node.js, Vanilla HTML/CSS/JS.

## Naming & Formatting Conventions
- **Project Naming**: All project names must be capitalized for each word, and connected with underscores (`_`). Example: `Stock_Price_Calculator`.
- **Variables & Code**: Generally follow standard JS conventions unless specified otherwise, but project/folder names strictly follow the `Word_Word_Word` underscore format.

## Design & Style Preferences
- **UI Theme**: "Studio Noir" (暗黑极简主义). `--bg: #030303`, `--fg: #ffffff`, `--fg-dim: #777777`. High-end, editorial, borderless aesthetic with heavy reliance on negative space and strict typographic alignment.
- **Layout**: Two-column layout with extreme spacing. The calculator form is on the left, and the history ledger is on the right.
- **Window Management**: 
  - **Window Dimensions**: Default initial size is extremely tall and narrow (600x1180) to force a "Studio Noir" vertical split layout that feels like a physical receipt or poster. Window State Persistence is fully supported and enabled.
  - Floating above other apps (`alwaysOnTop: true`).
- **Icon Style**: Minimalist Flat Apple Style (like modern iOS flat design) with exact 15% transparent padding.

## Current Status & Features
- **Current Version**: v4.0.0 (Master branch). Legacy versions maintained on `v2.2.x`, `v2.3.x`, and `v3.0.0`.
- **Key Features**:
  - **Custom Labels System (v4.0.0)**: A new pill-shaped custom labels module in the left panel. Supports inline editing, drag-to-reorder, and a drag-to-trash deletion zone.
  - **Drag-to-Replace Titles**: Seamlessly drag any custom label and drop it directly onto any right-panel history record's title to instantly replace it.
  - **Silent DOM Syncing**: All drag-and-drop ordering (both tags and main history groups) is now synced via underlying vanilla JS DOM manipulation to completely eliminate visual flickering while preserving global data synchronization.
  - **Advanced Group Header Interactions**: The right-panel group header separates interactions cleanly. Single-click the stock code to edit the identifier. Single-click the stock name to edit the display name. Double-click the stock name to jump to the TradingView web chart.
  - **Real-Time News Feed**: Integrated real-time EastMoney API fetching via Electron IPC to display stock-specific latest announcements and news, maintaining the Studio Noir layout.
  - **Window Size Toggle**: A top-right SVG button uses `preload.js` and IPC to seamlessly toggle the Electron window between the default "poster" layout (600x1180) and the user's previously dragged custom dimensions.
  - **Light/Dark Mode Toggle**: Persistent theme switching via a minimalist top-right icon.
  - **Studio Noir Redesign**: Masthead currency toggles, borderless aesthetic, unified typography (1.2rem for values, 0.75rem for labels).
  - **History Ledger Grid**: History records are rendered into a strict, perfectly aligned CSS grid with dynamic parsing for legacy text-based records. Visual separation between stock groups is achieved via extreme whitespace (4rem bottom margin).
  - **Inline Shares Recording (v2.2.0+)**: Minimalist inline input field directly on the history record to log share counts. Auto-saves and persists seamlessly without native UI spinners.
  - **Multi-Timeframe Analysis Memo (v2.3.0+)**: Group notes are enhanced with three responsive timeframe input fields (`W:`, `D:`, `30:`) for technical analysis, positioned neatly above the traditional open text note area. All fields auto-save, survive drag-and-drop rearrangements, and export accurately to HTML reports.
  - **History Grouping**: History records are grouped automatically by stock symbol.
  - **Drag-and-Drop**: Items can be sorted within their groups, and entire groups can be rearranged vertically. Live input fields gracefully retain data during sorting.
  - **Layout & Protection**: A dedicated absolute-fixed draggable top region protects macOS window controls.
  - Reverse calculation: Base and Target prices can reverse-calculate each other.
  - **Draggable Splitter**: The middle gap is dynamically draggable to reallocate width between the control panel and history ledger. Absolute min-width bounds are strictly enforced (235px left, 281px right) to prevent layout breakages. Custom layouts are persisted to localStorage.
  - **Inline Editing**: Users can click the label values inside history records to seamlessly open an inline input box, edit the base price, target, percentages or movement directions, and auto-recalculate everything without modals.
  - Input auto-select: Clicking any text or number input auto-selects its content.
